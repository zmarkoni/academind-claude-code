NextNotes Project Accessibility Guidelines & Tools

Overview

NextNotes implements comprehensive accessibility (a11y) testing and enforcement to ensure all users can navigate and use the application, regardless of their abilities. This document outlines the tools, setup, and best practices in place.

✓ Status: ESLint accessibility plugin installed and configured. Axe DevTools integration for Playwright e2e tests ready. Heading hierarchy validation implemented.

Tools Installed & Configured

1. eslint-plugin-jsx-a11y

Package: eslint-plugin-jsx-a11y
Purpose: Lints React/JSX code for accessibility violations during development
Installed: Yes ✓

Automatically catches accessibility issues in your components:

  • Empty headings and labels
  • Missing alt text on images
  • Missing ARIA roles
  • Interactive elements without proper semantics

Enabled Rules:

{
  "jsx-a11y/heading-has-content": "warn"
}

2. @axe-core/playwright

Package: @axe-core/playwright
Purpose: Automated accessibility testing in Playwright e2e tests
Installed: Yes ✓

Runs accessibility scans during e2e testing to catch issues before deployment:

  • WCAG 2.1 compliance checking
  • Heading hierarchy validation
  • Color contrast issues
  • Form accessibility
  • Screen reader compatibility

Usage in Tests:

import { AxeBuilder } from "@axe-core/playwright";

test("header passes accessibility scan", async ({ page }) => {
  await page.goto("/authenticate");
  const results = await new AxeBuilder({ page })
    .include("header")
    .analyze();
  expect(results.violations).toHaveLength(0);
});

Implementation in This Project

ESLint Configuration

File: eslint.config.mjs

import { defineConfig } from "eslint/config";
import nextVitals from "eslint-config-next/core-web-vitals";
import nextTs from "eslint-config-next/typescript";

const eslintConfig = defineConfig([
  ...nextVitals,
  ...nextTs,
  {
    rules: {
      "jsx-a11y/heading-has-content": "warn"
    }
  }
]);

This configuration runs automatically on every bun run lint and via the PostToolUse hook.

E2E Accessibility Tests

File: tests/e2e/accessibility.spec.ts

Tests included:

  • Axe Scans: Full page and component-level accessibility audits
  • Heading Hierarchy: Validates h1→h2→h3 progression (no skips like h2→h4)
  • Content Validation: Ensures all headings have accessible text content
  • Multi-page Testing: Checks both authenticated and unauthenticated pages

Run Accessibility Tests:

# Run only accessibility tests
bun run test:e2e tests/e2e/accessibility.spec.ts

# Run all e2e tests (includes accessibility)
bun run test:e2e

Development Workflow

During Development

  1. Edit/create components with proper semantic HTML (<button>, <nav>, <header>, etc.)
  2. ESLint plugin automatically warns about accessibility violations
  3. bun run lint --fix to auto-fix where possible

Before Committing

  1. Run linter: bun run lint (eslint-plugin-jsx-a11y warnings included)
  2. Run e2e tests: bun run test:e2e (accessibility.spec.ts included)
  3. Fix any accessibility warnings or test failures

Continuous Integration

Both linting and e2e tests should run in your CI/CD pipeline to prevent accessibility regressions.

  <article>...</article>

<h2>Section</h2>
<h3>Subsection</h3>


<h1>Page Title</h1>
<h3>Subsection</h3> <!-- Should be h2 -->

Images & Alt Text


<img src="logo.svg" alt="NextNotes application logo" />


<img src="logo.svg" alt="image" />
<img src="logo.svg" />

Form Accessibility


<label htmlFor="email">Email</label>
<input id="email" type="email" />


<label>Email</label>
<input type="email" />

ARIA & Color Contrast

  • Use ARIA attributes only when semantic HTML isn't sufficient
  • Maintain 4.5:1 contrast ratio for normal text, 3:1 for large text
  • Don't rely on color alone to convey information

WCAG 2.1 Compliance

This project aims for WCAG 2.1 AA compliance, covering:

  • Perceivable: Information is visible/readable
  • Operable: All functionality is keyboard accessible
  • Understandable: Content is clear and predictable
  • Robust: Works with assistive technologies

Troubleshooting

ESLint Warning: "heading-has-content"

Issue: Your heading element is empty or has no accessible text.

Solution: Ensure all headings have content:


<h2></h2>


<h2>Section Title</h2>

Axe Test Failure: Heading Hierarchy

Issue: Heading levels skip (e.g., h2 → h4 without h3).

Solution: Follow sequential heading levels:


<h2>Main</h2>
<h4>Sub</h4> <!-- Missing h3 -->


<h2>Main</h2>
<h3>Sub</h3>
<h4>Sub-sub</h4>

Axe Test Failure: Color Contrast

Issue: Text color doesn't have sufficient contrast against background.

Solution: Use tools like WebAIM Contrast Checker to verify 4.5:1 (normal) or 3:1 (large text) ratio.
