🎣 Hooks Configuration

Claude Code PostToolUse Hooks in NextNotes

Overview

Hooks are automated actions that run in response to specific events in Claude Code. In NextNotes, we use PostToolUse hooks to automatically format and lint code after every edit.

✓ Status: PostToolUse hooks are configured and active. They run Prettier (formatter) and ESLint (linter) on every file change.

What Are Hooks?

Hooks are rules defined in .claude/settings.json that automatically execute shell commands in response to Claude Code events.

Hook Types:

  • PostToolUse - Runs after a tool is executed (Edit, Write, Bash, etc.)

They help maintain code quality without requiring manual intervention.

Current Hooks Setup in NextNotes

File: .claude/settings.json

PostToolUse Hook Configuration

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd \"$CLAUDE_PROJECT_DIR\" && npx prettier --write \"$CLAUDE_CHANGED_FILES\" 2>/dev/null && eslint --fix \"$CLAUDE_CHANGED_FILES\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
Matcher: Edit|Write - Hook runs after Edit or Write tool use
Command: Runs Prettier then ESLint on changed files
Error Handling: || true prevents hook failure from blocking operations

Hook Execution Flow

When You Edit a File

1. You use Edit tool on a file
2. Edit completes successfully
3. PostToolUse hook triggers
4. Prettier formats the file
5. ESLint fixes linting issues
6. File is updated with formatting/linting changes
7. You see the updated file

Example: Adding h3 Element

// Before hook runs:
      <h2>Title</h2>
              <h3>TEST</h3>  // Bad indentation

// After Prettier:
      <h2>Title</h2>
      <h3>TEST</h3>  // Fixed indentation

// After ESLint:
      <h2 className="sr-only">Title</h2>
      <h3>TEST</h3>  // Fixed class name and other issues

Tools Executed by Hooks

1. Prettier (Code Formatter)

What it does: Automatically formats code
Formats: JavaScript, TypeScript, JSX, CSS, JSON, Markdown, YAML
Focus: Whitespace, indentation, line length
Command: npx prettier --write "$CLAUDE_CHANGED_FILES"

What Prettier Fixes:

  • Indentation (tabs vs spaces)
  • Line length and wrapping
  • Quote styles (' vs ")
  • Semicolons
  • Trailing commas
  • JSX formatting
Prettier Official →

2. ESLint (Code Linter)

What it does: Identifies and fixes code quality issues
Checks: JavaScript/TypeScript syntax, best practices, accessibility
Fix Flag: --fix auto-corrects fixable issues
Command: eslint --fix "$CLAUDE_CHANGED_FILES"

What ESLint Fixes:

  • Unused variables
  • Incorrect imports
  • Accessibility violations (via jsx-a11y plugin)
  • React hooks rules
  • Next.js specific patterns
  • TypeScript type issues
ESLint Official →

ESLint Plugins Active:

  • eslint-config-next - Next.js best practices
  • eslint-plugin-jsx-a11y - Accessibility checks

Manual Commands (Without Hooks)

If hooks don't run or you need manual control, use these commands:

Format with Prettier

bun exec prettier --write .

Lint and Fix with ESLint

bun run lint --fix

Or run through npm/bun

bun run lint

Troubleshooting Hooks

Hook Didn't Run?

Possible Causes:

  • Hook only runs on Edit and Write tools
  • Bash commands don't trigger hooks (by design, to avoid infinite loops)
  • Check .claude/settings.json for configuration issues

Prettier Installation Failed

Solution: Install Prettier manually with:

bun add -d prettier

ESLint Not Found

Solution: Check that ESLint is installed. Should already be in devDependencies. If not:

bun add -d eslint

Hook Error Hiding

The hook ends with || true which silently ignores errors. This prevents tool use from failing if formatting fails. To see errors: 

bun exec npx prettier --write "components/header.tsx"
bun run lint --fix

Full Settings Configuration

File Path: .claude/settings.json 

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd \"$CLAUDE_PROJECT_DIR\" && npx prettier --write \"$CLAUDE_CHANGED_FILES\" 2>/dev/null && eslint --fix \"$CLAUDE_CHANGED_FILES\" 2>/dev/null || true"
          }
        ]
      }
    ]
  },
  "enabledPlugins": {
    "typescript-lsp@claude-plugins-official": true
  }
}

Explanation:

  • "matcher": "Edit|Write" - Hook runs on both Edit and Write tools
  • "type": "command" - Execute shell command
  • cd "$CLAUDE_PROJECT_DIR" - Change to project root
  • npx prettier --write - Format all changed files
  • 2>/dev/null - Suppress error messages
  • || true - Don't fail hook if commands fail

Adding Additional Hooks

Example: Run Tests After Changes

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd \"$CLAUDE_PROJECT_DIR\" && bun run test 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}

Example: Custom Linter

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "cd \"$CLAUDE_PROJECT_DIR\" && bun run lint:strict 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
